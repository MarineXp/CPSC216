How It Works

Figure 1: System Diagram

The system design overview, as seen in Figure 1, consists of two parts: the left side being the main outside system and the right for the user interface. Each sensor's inputs and the 9-volt rechargeable battery supply are on the left side. The main system Arduino serves as the central microcontroller for the system. The main Arduino is responsible for interfacing with the sensor, data processing, and controlling the watering system. This enables reading sensor inputs, data conversion, and algorithm execution. The watering system depends on the user's need to implement a DC pump or a solenoid water valve. If using a solenoid valve, it is required to be powered by a 12-volt power supply. Wireless connection between the main system Arduino and the user interface Arduino is achieved through Bluetooth modules to communicate for remote control and monitorization. The code and calculations node establishes communication between sensors and output devices to record sensor data over time and create visual trends for user insight. Processing this data determines the timing and duration of watering cycles. In the user interface, the input is what the user manually adjusts settings, such as watering schedules or sensor thresholds. The user interface output is the displayed data and real-time sensor readings.
